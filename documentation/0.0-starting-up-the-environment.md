# 0.0 Starting Up the Environment

## Overview

This project runs entirely inside Docker. The stack is composed of 4 containers:

| Container | Technology | Role |
|---|---|---|
| `sc_db` | PostgreSQL 17 | Database |
| `sc_backend` | Spring Boot 3.5 + Java 21 | REST API |
| `sc_frontend` | Angular 21 + Nginx | SPA served as static files |
| `sc_nginx` | Nginx | Reverse proxy — the only public entry point |

All containers communicate through a shared internal Docker network (`sc_network`). The only port exposed to the outside world is `80` (Nginx).

```
Browser → localhost:80 → Nginx → /api/*  → Spring Boot :8080
                                → /*      → Angular     :80
```

## Prerequisites

### 1. Docker

Download and install **Docker Desktop** from the official site:

- **Linux:** https://docs.docker.com/engine/install/
- **Windows/Mac:** https://www.docker.com/products/docker-desktop/

Verify the installation:

```bash
docker --version
docker compose version
```

Expected output:
```
Docker version 27.x.x
Docker Compose version v2.x.x
```

> Docker Compose v2 is required. It comes bundled with Docker Desktop. On Linux, make sure you install the `docker-compose-plugin` package.

---

### 2. Node.js + Angular CLI

Node.js is required only to generate the Angular project. After that, everything runs inside Docker.

Download **Node.js LTS** from: https://nodejs.org

Verify:

```bash
node --version   # should be 22.x.x
npm --version    # should be 10.x.x
```

Install the Angular CLI globally:

```bash
npm install -g @angular/cli
```

Verify:

```bash
ng version
```

> You only need Node.js and the Angular CLI on your machine once — to scaffold the project. Day-to-day development runs inside Docker.

## First Time Setup

### Step 1 — Clone the repository

```bash
git clone <repository-url>
cd storage-control
```

### Step 2 — Create the environment file

```bash
cp .env.example .env
```

Open `.env` and review the values. For local development the defaults work fine, but **never use these values in production**.

```env
POSTGRES_DB=storage_control
POSTGRES_USER=scuser
POSTGRES_PASSWORD=changeme123

SPRING_DATASOURCE_URL=jdbc:postgresql://db:5432/storage_control
SPRING_DATASOURCE_USERNAME=scuser
SPRING_DATASOURCE_PASSWORD=changeme123

JWT_SECRET=dev_secret_replace_me_with_256bit_random_string
JWT_EXPIRATION_MS=86400000

JAVA_OPTS=-Xms256m -Xmx512m
```

> The `.env` file is listed in `.gitignore` and must never be committed to version control.

### Step 3 — Build and start the containers

```bash
docker compose up --build
```

The first run takes several minutes because Docker needs to:
- Download all base images (Maven, Node, Nginx, PostgreSQL)
- Download all Maven dependencies (~200MB)
- Download all npm packages
- Compile the Spring Boot app into a JAR
- Build the Angular production bundle

Subsequent runs are much faster due to Docker layer caching.

### Step 4 — Verify everything is running

```bash
docker compose ps
```

All 4 containers should show as running:

```
NAME           STATUS
sc_db          running (healthy)
sc_backend     running
sc_frontend    running
sc_nginx       running
```

### Step 5 — Open the app

Open your browser and go to:

```
http://localhost
```

You should see the Angular welcome page.

## Daily Development

After the first setup, you don't need `--build` unless you change a `Dockerfile` or `pom.xml`/`package.json`.

| Command | When to use |
|---|---|
| `docker compose up` | Start all containers (uses cached images) |
| `docker compose up --build` | Rebuild images then start (use after Dockerfile changes) |
| `docker compose down` | Stop and remove containers |
| `docker compose down -v` | Stop containers and delete database volume |
| `docker compose logs -f` | Watch live logs from all containers |
| `docker compose logs -f backend` | Watch logs from one container only |
| `docker compose ps` | Check status of all containers |

## Port Reference

| Service | Host port | Notes |
|---|---|---|
| Nginx (app) | `80` | Main entry point — open this in the browser |
| PostgreSQL | `5433` | Only for connecting with a DB client (DBeaver, psql) |
| Backend | not exposed | Only reachable through Nginx internally |
| Frontend | not exposed | Only reachable through Nginx internally |

> PostgreSQL uses host port `5433` (not the default `5432`) to avoid conflicts with any local PostgreSQL installation.

## Connecting to the Database (optional)

To inspect the database with a client like **DBeaver** or **TablePlus**:

| Field | Value |
|---|---|
| Host | `localhost` |
| Port | `5433` |
| Database | `storage_control` |
| Username | `scuser` |
| Password | `changeme123` |

Or via the terminal:

```bash
docker exec -it sc_db psql -U scuser -d storage_control
```

## Troubleshooting

### Variables not set warnings
```
WARN: The "POSTGRES_PASSWORD" variable is not set.
```
The `.env` file is missing or in the wrong directory. Make sure it exists in the project root (same folder as `docker-compose.yml`) and has all variables filled in.

### Port already allocated
```
Bind for 0.0.0.0:5432 failed: port is already allocated
```
You have a local PostgreSQL running on port 5432. Change the host port in `docker-compose.yml` under the `db` service:
```yaml
ports:
  - "5433:5432"
```

### Backend keeps restarting
The database might not be ready yet. Spring Boot waits for the `db` healthcheck to pass before starting. Wait 30 seconds and check:
```bash
docker compose logs backend
```

### Blank page in browser
The Angular build output path may not match the Dockerfile. Verify the path inside `frontend/Dockerfile`:
```dockerfile
COPY --from=builder /app/dist/frontend/browser /usr/share/nginx/html
```
